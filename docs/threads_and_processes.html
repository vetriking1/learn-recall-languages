🔀Multithreading & Multiprocessing ★ concept focus

Threads share memory inside one process; processes run separately with their own memory — spawn, join, lock, and launch.

python/17_threads_and_processes.py typescript/17_threads_and_processes.ts csharp/17_threads_and_processes.cs rust/17_threads_and_processes.rs cpp/17_threads_and_processes.cpp java/17_threads_and_processes.java
Thread vs process: threads are lighter and share memory (needs a lock/mutex for safe counters). Processes are isolated — use them when you need a separate program or crash isolation. async/await (topic 16) is cooperative single-threaded concurrency; this topic is preemptive parallelism.

Recall: Python GIL limits CPU parallelism in threading — use multiprocessing for CPU-bound work. I/O-bound work can use threads or async.
C++ / Rust threads need a full threading toolchain (MinGW-w64 or MSVC for C++; --edition 2021 for Rust).
