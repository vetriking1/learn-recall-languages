📐Abstraction

Defining a contract (what), leaving out the how.

Two tools, one idea: an abstract class (can hold state + some concrete code) vs an interface / trait (pure contract). You can never instantiate either directly — a concrete type must fill in the missing methods.

Recall: C++ has no abstract keyword — one pure-virtual method (= 0) makes the class abstract. Rust folds "interface" and "abstract class with shared code" into a single concept, the trait (required methods + optional defaults). Python enforces it at runtime; the statically-typed languages catch it at compile time.
