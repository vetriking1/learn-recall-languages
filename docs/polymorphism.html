🎭Polymorphism

One interface, many types — the right method runs.

What does the collection hold? Dynamic dispatch needs an indirection: a base reference (Python/TS/Java/C#), a base pointer (unique_ptr<Shape> in C++, never by value — that "slices"), or a trait object (Box<dyn Shape> in Rust).

Mixed collection [Circle(2), Rect(3,4)] Shape[] List<Shape> Vec<Box<dyn Shape>> vector<unique_ptr<Shape>> List<Shape>

Needs virtual? no (duck typing) no (default) yes (virtual/override) via trait method yes (virtual) no (default)
Common base required? no (structural / duck) no (structural) yes (nominal) shared trait yes (nominal) yes (nominal)
Dispatch kind runtime runtime runtime (vtable) static (generics) or dyn runtime (vtable) runtime (vtable)
Recall: Python and TypeScript don't need a shared base at all — any object with the right methods works (duck / structural typing). C#, C++ and Java are nominal: the type must declare it extends/implements the base. Rust is unique in offering a zero-cost static option (generics) alongside runtime dyn dispatch.
