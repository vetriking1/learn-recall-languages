🧰Libraries & frameworks

Syntax topics (01–23) teach the language. This page maps each language to what teams actually ship with: web stacks, ORMs, test tools, and where the ecosystem is strongest.

Framework vs library: a library you call (requests, serde). A framework calls you (Django, Spring, ASP.NET) — it owns the structure; you fill in hooks.

Per-language ecosystem snapshot

Python Shines: fastest path from idea to working API or notebook; unmatched ML/data library depth.
Reach for: FastAPI (APIs), Django (batteries-included web), pandas (tables), pytest (tests).
Skip for: hard real-time, mobile-native UI, teams that need compile-time guarantees on every line.
TypeScript Shines: one typed language across browser + server; npm ecosystem size.
Reach for: React (UI), Next.js (full-stack), NestJS (structured API), Zod (validation).
Skip for: number-crunching kernels, kernel drivers, projects with no JavaScript runtime allowed.
C# Shines: productive OOP + LINQ + first-class tooling in Visual Studio / Rider.
Reach for: ASP.NET Core (web), EF Core (SQL), Unity (games), MAUI (cross-platform UI).
Skip for: tiny embedded firmware, teams standardized on JVM only.
Rust Shines: safe systems programming with cargo; WASM and CLI tools with single static binaries.
Reach for: tokio (async), serde (JSON), axum (web), clap (CLI).
Skip for: hackathon MVPs, shops without time for borrow-checker onboarding.
C++ Shines: control over memory and hardware; massive existing code in games and finance.
Reach for: STL, Boost (selectively), Qt (desktop), Unreal (games), CMake (build).
Skip for: new CRUD web apps unless you must integrate with C++ libs.
Java Shines: decade-proven server patterns, JVM monitoring, gigantic hiring pool in enterprise.
Reach for: Spring Boot (everything), Hibernate (ORM), Kafka clients, JUnit (tests).
Skip for: one-file scripts, front-end work (use TS/Kotlin for Android UI).
